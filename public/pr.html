Pull Request (PR)

A PR is used in the git flow when you want to collaborate into a project.

Is a better way than a simple merge to a repository, because that way your team can pre-check your work before you do any change/merge into a repo branch.

At least one person of your team must review your changes, and can tell you if it's ok or if something must be changed.

When everything is ok and your code is accepted, then the merge can be done, and that way nothing goes wrong with it (theoretically).

You can learn more in the github documentation in the github documentation.
